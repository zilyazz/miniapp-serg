# Supabase Changes

Сюда складываются SQL-файлы, которые нужно вручную проливать в Supabase.

Рекомендуемый порядок:

1. Открываете следующий по дате SQL-файл.
2. Копируете его в SQL Editor Supabase.
3. Проливаете.
4. После этого деплоите backend-код, который зависит от этих изменений.

Текущие файлы:

- `2026-04-22-vk-auth.sql` — добавляет поддержку VK-авторизации в общей таблице `users`.
- `2026-04-22-vk-votes-prices.sql` — добавляет отдельное поле `price_vk_votes` для цен на голоса VK.
